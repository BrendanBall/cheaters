Format for submission file
Submissions need to be uploaded as a zip file. The root of this zip file should contain folders named as users. Each folder should contain the source files belonging to that specific user. An example is shown below.

Assignment1.zip
— John/
—— Question1.py
—— Question2.py
— Mary/
—— Question1.py
—— Question2.py


Parameters
Parameter Default Description
Description Compulsory field Something which will help you identify this submission at a later stage. This is the only required field.
Compare from year None If you are comparing to past submissions, only submissions which were submitted on this year or later will be considered. Leaving this blank means no history comparisons will be made.
t 10 If there is a substring match at least as long as t, then this match is guaranteed to be detected. Keep in mind that function and variable names will be replaced and counted as one character.
n 6 No matches shorter than the n will be detected.
Min lines matched 3 The minimum number of consecutive lines which need to match before the set is flagged.
Separation allowance 2 Sometimes there will be large matching blocks of code. If there is one line in the middle of the block somewhere, it will split the block in two. This parameter controls how many lines within a block are allowed to not match, without splitting the larger block.
Match threshold 10 This is a percentage. All matches smaller than this will be discarded.
